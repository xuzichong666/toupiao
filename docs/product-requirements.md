# 共识投票小程序产品需求文档

## 1. 产品概述

**产品名称**：共识投票

**产品标语**：小分歧，大共识

**产品定位**：面向家庭、朋友、同学、同事等小范围社交场景的轻量投票工具。用户可以快速发起投票，邀请他人参与选择，并通过实时结果减少讨论成本、帮助大家达成共识。

**核心价值**：

- 快速发起：1 分钟内创建一个投票。
- 简单参与：点击选项即可表达想法。
- 结果清晰：实时统计票数、占比和领先选项。
- 达成共识：用投票减少争执，辅助群体决策。

**典型场景**：

- 周末去哪玩
- 晚餐吃什么
- 看哪部电影
- 去谁家聚会
- 活动时间、地点、方案选择

## 2. 用户与目标

**目标用户**：

- 需要在朋友、家庭或小群体中做选择的人。
- 希望用低成本方式收集意见的人。
- 不需要复杂问卷，只需要快速决策的小范围组织者。

**用户角色**：

- 发起人：创建投票、分享投票、查看结果、管理投票。
- 参与人：通过链接、卡片或列表进入投票，选择选项并提交。

**成功标准**：

- 用户能在首页快速找到“发起新投票”入口。
- 用户能完成创建、分享、参与、查看结果的完整流程。
- 参与人无需学习即可完成投票。
- 结果页能清楚表达每个选项的票数、占比和当前领先项。

## 3. 核心流程

### 3.1 发起投票流程

1. 用户进入首页。
2. 点击“发起新投票”。
3. 填写投票标题。
4. 添加至少 2 个投票选项。
5. 可选配置更多设置：
   - 是否允许多选
   - 是否匿名投票
   - 截止时间
   - 谁可以参与
6. 点击“创建”。
7. 进入“投票已创建”页面。
8. 选择分享方式邀请他人参与。

### 3.2 参与投票流程

1. 用户通过投票链接、投票卡片或分享入口进入投票详情。
2. 查看投票标题、发起人、剩余时间、投票规则。
3. 选择一个或多个支持的选项。
4. 点击“提交投票”。
5. 提交成功后进入实时结果页。
6. 可继续分享投票给更多人。

### 3.3 查看结果流程

1. 用户进入结果页。
2. 查看总参与人数。
3. 查看每个选项的票数、占比和进度条。
4. 查看当前最高票选项。
5. 已参与用户可分享投票。

## 4. 页面需求

### 4.1 首页：投票列表

**页面目标**：展示用户相关投票，并提供快速发起入口。

**主要模块**：

- 顶部品牌区：
  - 小程序名称：共识投票
  - 可保留微信小程序胶囊按钮位置
- 发起入口：
  - 卡片标题：发起新投票
  - 说明：快速创建投票，收集大家的意见
  - 图标：加号
- 筛选标签：
  - 全部
  - 我发起的
  - 我参与的
  - 已结束
- 投票卡片列表：
  - 状态标签
  - 投票标题
  - 发起人
  - 剩余时间或结束时间
  - 参与人头像组
  - 参与人数
  - 当前领先选项及占比
  - 进度条
- 底部导航：
  - 首页
  - 发现
  - 消息
  - 我的

**交互规则**：

- 点击“发起新投票”进入创建页。
- 点击投票卡片进入投票详情或结果页：
  - 进行中且未参与：进入投票详情。
  - 已参与：进入结果页。
  - 已结束：进入结果页。
- 切换筛选标签时刷新列表。
- 长按投票卡片打开更多操作菜单。

### 4.2 创建投票页

**页面目标**：让用户快速创建结构完整的投票。

**字段要求**：

- 投票标题：
  - 必填
  - 建议最多 30 字
  - 示例：周末去哪玩？
- 投票选项：
  - 至少 2 项
  - 每项必填
  - 支持继续添加选项
  - 可选支持选项图片
- 更多设置：
  - 允许多选：默认关闭
  - 匿名投票：默认开启
  - 截止时间：默认 24 小时后
  - 谁可以参与：默认所有人可参与

**按钮**：

- 左上：取消
- 右上：创建
- 表单内：添加选项

**校验规则**：

- 标题为空时不能创建。
- 有效选项少于 2 个时不能创建。
- 选项名称不建议重复。
- 截止时间必须晚于当前时间。

**创建成功后**：

- 进入“投票已创建”页。
- 显示创建成功状态。
- 提供分享入口。

### 4.3 投票已创建页

**页面目标**：确认创建成功，并引导用户分享邀请。

**展示内容**：

- 成功图标
- 文案：投票已创建
- 投票标题
- 说明：邀请朋友参与投票
- 已参与人数
- 参与人头像组

**分享入口**：

- 微信好友
- 微信群
- 复制链接
- 更多

**辅助入口**：

- 投票链接已复制提示
- 去粘贴给朋友

### 4.4 投票详情页

**页面目标**：让参与人完成投票。

**展示内容**：

- 关闭按钮
- 投票标题
- 投票类型标签：单选 / 多选
- 发起人
- 剩余时间
- 规则提示：
  - 单选：请选择你支持的选项
  - 多选：请选择你支持的选项，最多选 N 项
- 选项列表：
  - 选项图片
  - 选项名称
  - 当前票数
  - 进度条
  - 选中状态
- 已选数量提示：
  - 示例：已选 2/2
- 提交按钮
- 查看实时结果入口

**交互规则**：

- 单选投票只能选择 1 项。
- 多选投票可选择多项，并受最多选择数量限制。
- 未选择任何选项时，提交按钮不可用或提交时提示。
- 提交成功后进入实时结果页。
- 已结束、已取消投票不可再提交。

### 4.5 实时结果页

**页面目标**：展示投票统计，帮助用户判断共识。

**展示内容**：

- 投票标题
- 投票类型标签
- 发起人
- 剩余时间或结束状态
- 实时结果标题
- 参与人数
- 选项结果列表：
  - 选项名称
  - 票数
  - 占比
  - 进度条
- 投票已参与提示
- 分享投票按钮

**统计规则**：

- 占比 = 当前选项票数 / 总票数。
- 总票数为 0 时，所有选项占比显示 0%。
- 最高票选项需要视觉强调。
- 多个选项并列最高时，可同时视为领先。

### 4.6 更多操作菜单

**触发方式**：长按投票卡片。

**菜单项**：

- 复制链接
- 分享投票
- 编辑投票：仅发起人可见
- 结束投票：仅发起人可见
- 删除投票：仅发起人可见

**规则**：

- 非发起人不能编辑、结束、删除投票。
- 已结束投票不再展示“结束投票”。
- 删除操作需要二次确认。

## 5. 状态规则

### 5.1 投票状态

- 进行中：投票未结束，仍可参与。
- 已结束：投票已结束，无法修改投票选择。
- 已取消：投票被发起人取消。
- 即将结束：距离截止时间小于 30 分钟。

### 5.2 用户参与状态

- 未参与：可进入详情页选择并提交。
- 已参与：不可重复投票，可查看结果。
- 发起人：拥有编辑、结束、删除等管理能力。

### 5.3 结果可见规则

设计图倾向于“实时结果可查看”，但投票完成后会明确展示“投票已参与”。建议产品规则为：

- 默认允许查看实时结果。
- 如果后续需要防止结果影响选择，可新增“投票后可见”设置。

## 6. 数据需求

### 6.1 投票 Poll

- `id`：投票 ID
- `title`：投票标题
- `creatorId`：发起人 ID
- `creatorName`：发起人名称
- `type`：投票类型，单选或多选
- `anonymous`：是否匿名
- `allowMultiple`：是否允许多选
- `maxSelections`：最多可选数量
- `participantScope`：参与范围
- `deadline`：截止时间
- `status`：进行中、已结束、已取消
- `createdAt`：创建时间
- `updatedAt`：更新时间

### 6.2 选项 Option

- `id`：选项 ID
- `pollId`：所属投票 ID
- `text`：选项名称
- `imageUrl`：选项图片
- `votes`：票数
- `sortOrder`：排序

### 6.3 投票记录 Vote

- `id`：记录 ID
- `pollId`：投票 ID
- `userId`：用户 ID
- `optionIds`：选择的选项 ID 列表
- `createdAt`：提交时间

### 6.4 用户 User

- `id`：用户 ID
- `nickname`：昵称
- `avatarUrl`：头像

## 7. MVP 范围建议

### 7.1 第一版必须实现

- 首页投票列表
- 发起新投票
- 单选 / 多选投票
- 截止时间
- 参与投票
- 实时结果
- 已参与防重复投票
- 投票状态：进行中、已结束
- 本地或云端保存基础投票数据

### 7.2 第二版增强

- 微信好友 / 微信群分享
- 复制链接
- 参与人头像组
- 选项图片
- 长按更多操作菜单
- 编辑投票
- 删除投票
- 取消投票
- 即将结束状态

### 7.3 暂不纳入第一版

- 复杂问卷能力
- 评论区
- 排名投票
- 付费功能
- 公开投票广场
- 后台管理系统

## 8. 验收标准

- 用户可以从首页创建一个包含标题和至少 2 个选项的投票。
- 用户可以设置单选或多选。
- 用户可以在投票详情页选择选项并提交。
- 已参与用户再次进入同一投票时不能重复提交。
- 结果页能正确显示票数、百分比、进度条和最高票。
- 投票到达截止时间后不能继续提交。
- 首页能按全部、我发起的、我参与的、已结束筛选。
- 长按投票卡片能打开更多操作菜单。
- 发起人可以结束或删除自己创建的投票。
- 非发起人不能管理他人投票。

## 9. 与当前实现的差异

当前项目已具备基础的本地投票 MVP，但与设计图相比仍缺少：

- 投票创建成功后的分享邀请页。
- 首页底部导航、筛选标签和更完整的投票卡片信息。
- 截止时间、匿名投票、参与范围等高级设置。
- 选项图片、参与人头像组。
- 实时结果入口和分享投票能力。
- 长按卡片更多操作菜单。
- 编辑、结束、删除投票管理能力。
- 投票状态中的已取消、即将结束。
- 云端数据与跨设备真实参与。
